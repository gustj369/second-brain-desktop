# 용어 기준

## 데이터 구조
- **노트(note)**: brain.json 안의 단일 기록 단위. id·title·content·tags·type 필드 포함
- **링크(link)**: type="link"인 노트. URL 필드를 가지며 웹페이지 요약에 사용
- **태그(tag)**: 노트에 붙이는 키워드. 쉼표로 구분, tags 배열에 저장
- **brain.json**: 모든 노트 데이터를 저장하는 로컬 JSON 파일 (gitignore)
- **config.json**: API 키·설정을 저장하는 파일 (gitignore, 절대 커밋 금지)

## AI 기능
- **연결고리 찾기**: 현재 노트의 키워드 5개 + 연결 이유를 Gemini로 생성
- **비슷한 노트 찾기**: 현재 노트와 유사한 노트 3개를 Gemini로 추천
- **나에게 묻기**: 전체 노트를 컨텍스트로 질문 → 내 사고방식 기반 Gemini 답변
- **아이디어 충돌기**: 태그 비중첩 노트 2개 → 창의적 아이디어 3가지
- **일일 브리핑**: 앱 시작 시 랜덤 노트 3개 기반 Gemini 질문 팝업 (하루 1회)

## UI 컴포넌트
- **태그 클라우드**: 좌측 하단. 자주 쓴 태그일수록 크게 표시, 클릭 시 검색
- **복습 바**: 7일 이상 안 본 노트 열람 시 상단에 표시되는 초록 알림 바
- **토스트**: 클립보드 캡처 시 우측 하단에 8초간 표시되는 알림 팝업
- **지식 그래프**: 공통 태그로 연결된 노트 네트워크 시각화 창
- **관심사 창**: 월별 태그 빈도 차트 + GitHub 스타일 생산성 히트맵

## 개발 용어
- **바이브 코딩**: 코드를 직접 짜지 않고 AI와 대화로 만드는 방식
- **단계(Step/Phase)**: 기능 추가 단위. v1.0~v8.0까지 진행
- **원자 저장**: 임시 파일에 먼저 쓴 뒤 os.replace()로 교체하는 안전한 저장 방식
